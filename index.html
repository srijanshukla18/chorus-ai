The Problem

Every LLM is confidently wrong
about different things.

Claude hallucinates on X. GPT hallucinates on Y. Gemini gets Z wrong. You never know which model is wrong on which query. Single-model answers are a coin flip you can't see.

Chorus exploits this. By querying 5 fundamentally different models — different training data, different architectures, different countries of origin — and synthesizing the result, correlated failures cancel out. What remains is the signal.

This is LLM ensembling. The same principle that made Random Forests beat Decision Trees. Except now it's frontier AI models debating each other before answering you.

Grok's "Think" mode

4 copies of the same model running in parallel with different system prompts. Same training data, same architecture, same blind spots. When Grok is wrong, all 4 agents are wrong in the same way.

Chorus

5 different frontier LLMs — Claude, GPT, Gemini, MiniMax, Kimi — from 3 countries with 3+ architectures. Fundamentally different training data, different biases, different failure modes. Maximum decorrelation.

LLM ensembling isn't new. But doing it right means diversity, not duplication. 4 versions of the same model is a committee of clones. 5 genuinely different models is a jury of independent experts.

How It Works

Three steps. Zero servers.

01

Parallel Dispatch

Your prompt goes to all 5 models simultaneously via OpenRouter. One API key, one unified interface. All calls fire from your browser — no data touches any intermediary server.

02

Stream & Collect

Each model streams its answer in real-time. You can expand the "Agents thinking" panel to watch all 5 responses arrive simultaneously — the agreements, disagreements, and nuances.

03

Synthesize

A cost-efficient aggregator model reviews all 5 responses, resolves contradictions, flags disagreements, and produces a single high-confidence answer. Peer review, built into the loop.

Why This Works

Decorrelated errors cancel out.

Epistemic Diversity

5 companies. 3 countries. 3+ architectures. Dense, MoE, Sparse Attention. Western RLHF, Chinese RL-heavy training. Different data, different biases, different blind spots.

No Backend Lock-in

Your OpenRouter key. Your browser. API calls go directly from client to OpenRouter. No middleware, no data retention, no vendor lock-in. Fork it, self-host it, own it.

Marginal Cost, Not 5x

Claude + GPT + Gemini carry the cost. MiniMax at $0.30/M and Kimi at $0.45/M input keep the 4th and 5th models cheap. Ensemble diversity at marginal cost.

When It Matters

Not for "what's the weather." For high-stakes queries where being wrong costs real money: investment research, technical architecture, legal analysis, medical reasoning, critical code review.
